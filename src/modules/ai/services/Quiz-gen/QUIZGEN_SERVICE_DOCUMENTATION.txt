====QUIZ GENERATION SERVICE API DOCUMENTATION

Base URL: /ai
Authentication: Required (JWT Token in Authorization header)

====1. POST /ai/quizgen
====Description: Tao quiz questions bang AI va tra ve du lieu JSON da duoc validate
Method: POST
Role: Lecturer, Admin
Caller to LLM: quiz-generator

Request Body:
{
  "text": "Tao 5 cau hoi trac nghiem ve OOP cho sinh vien nam 2",
  "provider": "groq"
}

Request Fields:
  - text (required): string - Prompt de yeu cau AI tao quiz
  - provider (optional): 'gemini' | 'groq' | 'openai' - Provider uu tien muon thu truoc

Stream mode cho frontend:
  - Bat SSE bang query param: ?stream=true
  - Hoac gui header: Accept: text/event-stream
  - Neu khong bat stream, API van tra JSON nhu cu

Thought streaming contract:
  - Moi request stream se bat dau bang event progress co stage = sse.start
  - Backend se phat cac progress event theo tung buoc suy nghi / lay file / tao quiz / parse ket qua
  - Frontend co the map progress.stage thanh cac state UI tuong ung de hien thi thought steps

Progress stages de frontend adapt:
  - sse.start: da mo ket noi stream, dang xu ly yeu cau tao quiz
  - quiz.start: khoi tao luong tao quiz
  - quiz.file.context: da co san file context tu client
  - quiz.file.resolve: dang lay metadata file theo fileId
  - quiz.file.list: dang liet ke file trong class
  - quiz.file.match: da tim duoc file phu hop trong class
  - quiz.file.ready: da co file context va san sang dua cho LLM doc
  - quiz.generate: dang tao quiz theo cach thong thuong (khong file)
  - quiz.llm.generate: LLM dang sinh/noi suy noi dung quiz
  - quiz.parse: dang phan tich ket qua JSON tra ve
  - quiz.done: hoan tat tao quiz

SSE events:
  - progress: thong bao tien trinh suy nghi / lay file / tao quiz / parse ket qua
  - final: payload cuoi cung, van la quiz generation result
  - done: dong stream

Example SSE output:
  event: progress
  data: {"stage":"sse.start","message":"Da mo ket noi stream, dang xu ly yeu cau tao quiz..."}

  event: progress
  data: {"stage":"quiz.file.list","message":"Dang liet ke cac file trong lop de tim tai lieu phu hop...","data":{"classId":"L01"}}

  event: progress
  data: {"stage":"quiz.file.match","message":"Da tim thay file phu hop trong lop.","data":{"classId":"L01","fileId":"fid_123","fileName":"midterm_practice.pdf"}}

  event: progress
  data: {"stage":"quiz.llm.generate","message":"Gemini dang doc file va suy luan noi dung quiz..."}

  event: progress
  data: {"stage":"quiz.parse","message":"Dang phan tich ket qua quiz tu Gemini..."}

  event: final
  data: {"text":"Duoi day la 5 cau quiz duoc tao dua tren tai lieu...","questions":[{"content":"...","options_json":{"A":"...","B":"..."},"answer_key_json":{"correct":"A"},"points":1}],"provider":"gemini","rawText":"..."}

  event: done
  data: {"message":"Hoan tat stream tao quiz."}

Frontend note:
  - Neu muon visualize thought stream, nen map progress.stage sang 3 nhom UI:
    + Discovery: quiz.start, quiz.file.resolve, quiz.file.list, quiz.file.match
    + Generation: quiz.file.ready, quiz.generate, quiz.llm.generate
    + Completion: quiz.parse, quiz.done
  - Neu gui metadata.classId, frontend khong can tu lay danh sach file; backend se tu match file theo prompt.
  - Neu muon fix file cu the, co the gui metadata.fileId hoac metadata.fileContext.

Response Fields:
  - text: string - Loi giai thich ngan cua AI cho bo cau hoi
  - questions: Array<Question>
    - content: string (required) - Noi dung cau hoi
    - options_json: object | array (optional) - Cac lua chon dap an
    - answer_key_json: object | array (required) - Dap an dung
    - points: number (required in output, default = 1)
  - provider: 'gemini' | 'groq' | 'openai' - Provider da thanh cong
  - rawText: string - Noi dung text goc tra ve tu provider


====RETRY AND FALLBACK BEHAVIOR
====Description: Co che xu ly loi parse JSON trong QuizGenerationService

1. Service thu provider theo thu tu:
   - Neu request co provider: thu provider do truoc, sau do fallback qua provider con lai
   - Neu request khong co provider: thu theo thu tu mac dinh gemini -> groq -> openai

2. Moi lan goi provider:
   - Goi OuterApiService.chat(...)
   - Thu parse response thanh JSON object voi format bat buoc:
     {
       "text": "...",
       "questions": [ ... ]
     }

3. Neu provider tra text khong parse duoc JSON hoac sai schema:
   - Ghi log warning
   - Thu provider tiep theo

4. Chi throw loi khi tat ca provider deu that bai parse/validate hoac provider-level request fail


====VALIDATION RULES
====Description: Kiem tra schema cua response sau khi parse

Top-level JSON:
  - Phai la object
  - Co field "questions" va phai la array
  - Field "text" la optional string (neu khong co se la chuoi rong)
  - questions khong duoc rong

Moi phan tu question:
  - content: string, non-empty
  - answer_key_json: object | array | JSON string hop le (required)
  - options_json: object | array | JSON string hop le (optional)
  - points: number hop le neu co; neu khong co thi mac dinh = 1


====ERROR RESPONSES
====400 Bad Request
- text rong hoac thieu
- Khong parse duoc response thanh JSON
- JSON khong dung schema bat buoc
- Tat ca providers deu that bai parse/validate

Example:
{
  "statusCode": 400,
  "message": "Quiz generation failed for all providers. Details: groq: Quiz generation response is not valid JSON... | gemini: ...",
  "error": "Bad Request"
}

====401 Unauthorized
- Chua dang nhap
- JWT token khong hop le

====403 Forbidden
- User khong co role Lecturer/Admin

Example:
{
  "statusCode": 403,
  "message": "Access denied. Required roles: Lecturer, Admin. Your role: Student",
  "error": "Forbidden"
}


====NOTES
====General Information

0. JWT token gui trong Authorization header
   Format: Authorization: Bearer <token>

1. Endpoint nay da duoc bao ve boi JwtGuard + RolesGuard

2. Prompt he thong cho caller quiz-generator da ep model tra ve JSON object dung schema

3. Du lieu "rawText" giup debug khi can doi chieu output goc cua model

4. Validation va fallback duoc xu ly tai service layer, khong phu thuoc provider co bao loi hay khong

5. Khuyen nghi:
   - Prompt can ro rang ve so luong cau hoi, muc do kho, va chu de
   - Neu muon uu tien provider cu the, truyen field "provider"
