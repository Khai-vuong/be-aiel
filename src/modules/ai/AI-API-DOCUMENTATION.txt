====AI API DOCUMENTATION

Base URL: /ai
Authentication: Required (JWT Token in Authorization header for protected routes)

====1. POST /ai/chat
====Description: Chat tong hop qua orchestrator (co routing mode chat/quiz_assistant/insight)
Method: POST
Role: any (authenticated users)
Content-Type: application/json

Stream mode cho frontend:
  - Bat SSE bang query param: ?stream=true
  - Hoac gui header: Accept: text/event-stream
  - Neu khong bat stream, API van tra JSON nhu cu

SSE events:
  - progress: thong bao tien trinh (suy nghi, validate, execute, lay file, tong hop...)
  - final: response cuoi cung, payload chinh la AiResponseDto
  - done: thong bao dong stream

Request Body (AiRequestDto):
{
  "text": "Tao cho toi 5 cau hoi ve SQL",
  "metadata": {
    "classId": "L01",
    "courseId": "C01"
  },
  "conversationId": "conv_123",
  "provider": "groq",
  "temperature": 0.7,
  "instructionPrompt": "Ban la tro ly hoc tap",
  "serviceType": "STUDY_ANALYST"
}

AiRequestDto fields:
  - text: string (required)
  - metadata: object (optional, dung de truyen context cho orchestrator)
  - conversationId: string (optional, dung de tiep tuc hoi thoai)
  - provider: string (optional)
  - temperature: number (optional)
  - instructionPrompt: string (optional)
  - serviceType: enum (optional, test/future only)
    + SYSTEM_CONTROL
    + STUDY_ANALYST
    + TUTOR
    + TEACHING_ASSISTANT

Response Body (AiResponseDto):
{
  "success": true,
  "conversationId": "conv_123",
  "messageId": "msg_456",
  "text": "Day la cau tra loi cua AI",
  "metadata": {
    "serviceType": "Chat",
    "provider": "groq",
    "processingTime": 250
  }
}

AiResponseDto fields:
  - conversationId: string (optional)
  - conversationTitle: string (optional)
  - messageId: string (optional)
  - success: boolean (required)
  - text: string (required)
  - error.message: string (optional)
  - error.code: string (optional)
  - metadata.createdAt: string (optional)
  - metadata.serviceType: string (optional)
  - metadata.provider: string (optional)
  - metadata.processingTime: number (optional)

====2. POST /ai/chat/direct
====Description: Chat truc tiep qua outer API (bo qua orchestration mode)
Method: POST
Role: any (authenticated users)
Content-Type: application/json

Stream mode:
  - Hanh vi giong /ai/chat: dung ?stream=true hoac Accept: text/event-stream
  - Event final van tra AiResponseDto

Request Body:
  - Su dung AiRequestDto nhu /ai/chat

Example Output:
{
  "success": true,
  "conversationId": "conv_123",
  "messageId": "msg_999",
  "text": "Tra loi direct chat",
  "metadata": {
    "processingTime": 180
  }
}

====SSE RESPONSE FORMAT
Khi stream duoc bat, frontend se nhan duoc cac event co dang sau:

event: progress
data: {"stage":"rag.plan","message":"Dang suy nghi buoc 1/7 de chon cong cu phu hop...","data":{"loop":1}}

event: progress
data: {"stage":"rag.execute","message":"Dang thuc thi 2 cong cu: class-quizzes, get-file","data":{"capabilityIds":["class-quizzes","get-file"]}}

event: final
data: {"success":true,"conversationId":"conv_123","messageId":"msg_456","text":"Day la cau tra loi cua AI","metadata":{"serviceType":"Chat","provider":"groq","processingTime":250}}

event: done
data: {"message":"Hoan tat stream phan hoi."}

Frontend note:
  - Nen dung EventSource neu chi can receive stream 1 chieu
  - Neu dung fetch streaming, can parse SSE chunks theo tung dong event/data

====3. POST /ai/quizgen
====Description: Tao quiz va co the doc file de phat sinh cau hoi chinh xac hon
Method: POST
Role: Lecturer, Admin
Content-Type: application/json

Stream mode:
  - Bat SSE bang query param: ?stream=true
  - Hoac gui header: Accept: text/event-stream
  - Neu khong bat stream, API van tra JSON nhu cu

Request Body:
{
  "text": "Hay tao 5 cau quiz dua tren noi dung cua file A",
  "provider": "groq",
  "temperature": 0.7,
  "metadata": {
    "fileId": "fid_123"
  }
}

File input options:
  - metadata.fileId: string (khuyen nghi, backend se tu fetch metadata file)
  - metadata.fileContext: object { url, mime_type, filename } (neu frontend da co san URL)
  - metadata.fileUrl + metadata.fileMimeType + metadata.fileName: fallback neu muon gui rieng
  - metadata.classId: backend se list file trong class, match theo ten file duoc nhac trong prompt, roi fetch metadata

Frontend can chi gui:
  - text
  - metadata.classId
  - prompt co nhac toi file muon dung (vi du: "file midterm practice")

SSE events:
  - progress: dang resolve file, doc file, tao quiz, parse ket qua
  - final: payload cuoi cung, van la quiz generation result
  - done: dong stream

Example Output:
{
  "text": "Duoi day la 5 cau quiz duoc tao dua tren tai lieu...",
  "questions": [
    {
      "content": "...",
      "options_json": {"A": "...", "B": "..."},
      "answer_key_json": {"correct": "A"},
      "points": 1
    }
  ],
  "provider": "gemini",
  "rawText": "..."
}

====8. DELETE /ai/conversations/:id
====Description: Xoa 1 conversation cua user hien tai
Method: DELETE
Role: any (authenticated users)
URL Parameters:
  - id (required): string - Conversation ID

Example Output:
{
  "message": "Conversation deleted successfully"
}

====HOW TO GET conversationId
====Option A: Lay truc tiep tu response cua /ai/chat hoac /ai/chat/direct
  - conversationId duoc tra ve sau moi lan chat thanh cong.

====Option B: Lay tu danh sach /ai/conversations
  - Goi GET /ai/conversations de lay cac conversation id (acid).

====AUTHENTICATION
====JWT token phai duoc gui trong Authorization header
Format: Authorization: Bearer <token>

====NOTES
====1. Neu khong gui conversationId trong request chat, he thong tu tao conversation moi.
2. Neu gui conversationId, he thong tiep tuc hoi thoai cu.
3. serviceType runtime thuong la: Chat, quizgen, insight.
4. provider la nha cung cap AI duoc su dung de tao response.
5. processingTime la thoi gian xu ly (ms).
6. SSE progress chi la thong bao tien trinh, khong phai ket qua tam thoi.
7. Event final trong SSE la payload chinh xac nhu JSON response thong thuong.
